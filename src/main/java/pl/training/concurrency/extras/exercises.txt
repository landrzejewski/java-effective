Task 1
Design a mechanism/algorithm to synchronize bathroom use in a company, assuming the following conditions:
- It is impossible for a woman and a man to enter the bathroom
- No more than three people can be in the bathroom at the same time

Task 2
Write code to simulate the production of an H20 molecule
- Each atom is produced by a separate thread
- The program should wait until a full molecule can be created before starting to build another
- Threads can be created in any order and can produce atoms at any speed

Task 3
A hairdressing salon has N chairs for customers and one hairdresser (hours)
- If there are no customers, the hairdresser should be put to sleep
- If a customer enters the salon and there are free chairs, the customer sits down and waits their turn
- If a customer enters the salon and there are no free chairs, the customer resigns (leaves the shop)
- If the hairdresser is asleep, the customer wakes/activates them
Write a program to coordinate the interactions described

Task 4
Imagine you have a container that fills tokens at a rate of 1 per second.

The container can hold a maximum of N tokens. Implement a thread-safe class that allows threads to run when a token is available. If no token is available, threads requesting a token should be blocked. The class should provide a method called getToken, which blocks the thread until a token is available.

Task 5
Implement a blocking queue using:
- implicit locks (monitor)
- explicit locks
- semaphore

Task 6
Imagine that at the end of a political conference, Party X and Party Y are trying to leave and simultaneously
ordering taxi rides. To ensure that a fight doesn't break out during the ride, you should develop an algorithm
according to which a ride is possible when there are only politicians from one party
or exactly two from each Party X and Y in the car.
